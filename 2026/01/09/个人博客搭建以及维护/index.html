5.3、执行一键部署

在终端输入这串“终极命令”：

  • clean: 清除缓存。
  • g (generate): 生成静态 HTML 网页。
  • d (deploy): 上传到 GitHub。

关键点提示：

  1. 登录验证：在执行 hexo d 的过程中，电脑可能会弹出一个小窗口要求你登录 GitHub。请点击 “Sign in with your browser” 进行授权即可。
  2. 成功标志：当你看到终端最后显示 INFO Deploy done: git，说明上传成功了！
  3. 稍等片刻：GitHub Pages 渲染需要一点时间。上传后大约过 1 分钟，你就可以在浏览器访问：https://test.github.io。

6. 配置 Gitalk 评论系统

Gitalk 是一款基于 GitHub Issues 的评论插件。由于 Typography 主题可能未内置该功能，我们需要手动进行“代码注入”。

6.1 注册 GitHub OAuth Application

  1. 登录 GitHub，进入 Settings -> Developer settings -> OAuth Apps -> New OAuth App。
  2. 填写信息：
    • Application name: 随便填（如 MyBlogComments）。
    • Homepage URL: https://你的用户名.github.io。
    • Authorization callback URL: 必须填写你的博客域名 https://你的用户名.github.io。
  3. 注册后获取 Client ID 和 Client Secret（需点击生成），保存备用。

6.2 安装 MD5 插件（防止长链接报错）

GitHub Issue 的标签长度限制为 50 个字符。如果文章标题过长，Gitalk 会初始化失败。

  1. 下载 md5.min.js。
  2. 将其存放在：themes/typography/source/js/md5.min.js。

6.5 初始化评论

部署博客后，你（管理员）必须亲自访问每一篇新文章并点击“登录 GitHub”按钮进行初始化，随后读者的评论功能才会激活。

7. 常见问题处理

  • 部署失败 (Connection Reset)：GitHub 连接不稳定时，可在终端执行 git config --global http.proxy http://127.0.0.1:端口号 设置代理。
  • 样式不生效：每次修改 _config.yml 或主题模版后，建议养成执行 hexo clean && hexo g -d 的习惯。
